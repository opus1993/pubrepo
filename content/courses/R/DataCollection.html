Data Collection

(Add content for read.table, base-R read.csv, and read.excel)

Every project must start with a team-wide understanding of the client’s pain points and the decisions that they face. What knowledge gives your client a competitive advantage? What is their naiive baseline?

Given a firm grasp of the businesses expectations, start exploring available data resources. Be cognisant of how the data was acquired, and any biases. This crucial work normally already has practitioners embedded within the team of domain experts.

R offers packages for importing most sources of data, ranging from CSV text to databases to SAS. For basic formats, the most commonly used are the packages readr, data.table, and readxl

All packages must first be installed to R, via install.packages(" "). At the beginning of the R script, it then is available for loading into session with

library(readr)

As an example, we will read the Used Cars dataset from notredame.analytics.data

usedcars <-
  read_csv('./data/usedcars.csv')
## Parsed with column specification:
## cols(
##   year = col_double(),
##   model = col_character(),
##   price = col_double(),
##   mileage = col_double(),
##   color = col_character(),
##   transmission = col_character()
## )

Note the column specification assumed by the read_csv command. For data sets with more than 1000 lines, it is best to set col_types = to a character vector to force the field types. See ?read_csv for the help for the function.
